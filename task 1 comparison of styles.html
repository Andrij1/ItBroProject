Comparison of the XML-HTML-MD-JSON markup languages

Features XML HTML MD JSON
Meaning and main purpose eXtensible Markup Language - is used to create other markup languages, intended for documents published on the Web. HyperText Markup Language - the standard language for Web pages. MarkDown markup language - for better reading, conversion into HTML/XML structures. JSON is the JavaScript Object Notation and is the lightweight format for storing and delivering data.
Developer (year), info source John Bosak (1996), roseindia.net Tim Berners-Lee (1993), washington.edu John Gruber (2004), programminghistorian.org JDavid Crockford (2001), json.org
Main versions XML 1.0 (1991) – XML 1.1. II-nd ed. (2006). HTML – HTML 4.01 (2000). HTML 5.0 (2011) draft version. Markdown (2004) etc. JSON (ECMA, 2001) etc.
Main features Consists of markup (<,>,&,;) and content, has strict rules, interoperability, document types defining, tags <…> using, An attribute has a single value. Case-sensitive tags. Star-end tags match, cannot start ("-", "." ) or contain (!"#$%&'()*+,/;<=>?@[\]^`{|}~,space) a list of characters. Supports headings, subheadings, paragraphs, addresses, anchors, lists, images, tables, styles (CSS), scripts (JS), media (audio/video), invisible canvas for a page markdown. Uses container (open, /close) and empty (open /) tags. Easy for reading and typing, doesn’t use tags, not many special symbols, fast rendering, isn’t strict, interoperable and convertible into other codes. Easily red by both – the man and the machine, applied to better interchange info between PCs, is vulnerable for viral content unless special measures of protection are undertaken, has close to JS coding – uses [], {}, colons, and commas to arrange values that are strings or numbers into the two basic structures.
Main differences Strict code, for data storage/interchange, does not contain text, except for code elements. Not strict–txt allowed, severe markup errors not fatal, unsuitable for data storage. Convenient for interchange of info, easily readable, plain text formatting for writing in the web. Easily readable, low character number, quick in rendering.
Main application by realms Used in data interchange, office tools etc. The only language used for creating web-pages. Can be used for inclusion into other codes. Widely used on web applications for data interchange.
